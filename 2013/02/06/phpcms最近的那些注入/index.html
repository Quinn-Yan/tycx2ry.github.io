phpcms最近的那些注入

发表于 2013-02-06 | 更新于: 2018-02-26 | 分类于 0day漏洞

    phpcms v9在过年的时候很火啊！很是出了几个洞。这个可能要从乌云说起。

第一个：

    http://www.wooyun.org/bugs/wooyun-2013-016984

    http://www.wooyun.org/bugs/wooyun-2013-017032

    http://www.wooyun.org/bugs/wooyun-2013-017119

这3个是乌云的三连发，作者有具体详细的分析了，我也就不多做解释了。

    然后又是t00ls上的一个注入，这个注入和乌云第二个其实都是利用了

&nbsp;&nbsp;public function add_special_char(&amp;`$value) `

这个函数过滤不严而造成的。

这儿有一个详细分析:http://n3w1yb1r7h.sinaapp.com/?p=15

 

其实 add_special_char函数过滤不严在1月10号的补丁给出了修补如下：

public function add_special_char(&$value) {
 if(’*’ == $value || false !== strpos($value, ’(’) || false !== strpos($value, ’.’) || false !== strpos ( $value, ’`’)) {
 //不处理包含* 或者 使用了sql方法。
 } else {
 $value = ’`’.trim($value).’`’;
 }
 if (preg_match("/(select|insert|update|delete)/i", $value)) {
 $value = preg_replace("/(select|insert|update|delete)/i", ’’, $value);
 }
 return $value;
 }

这个补丁没有补严密，依然有绕过的可能：

用/&#42!50000select&#42/就可以绕过检测了。

 

其实这个函数是一个相当关键的函数哦！

我们何不反查这个函数出现在那些语句中呢？

这样不是注入滚滚来了….
