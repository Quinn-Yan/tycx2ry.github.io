eval injection

发表于 2011-12-26 | 更新于: 2018-02-26 | 分类于 PHP|MYSQL学习

在90sec的论坛看见有人问eval injection的问题！

<?php    

echo ’<p>&#42——————————-&#42<p>’; 

echo ’This one can GET :<p><tb>http://192.168.174.147/test/eval_injection.php?arg=10;%20system%28%27/bin/ls%20/tmp%27%29’; 

$myvar = ’somevalue’; 

$x = $_GET[’arg’]; 

echo $x.”<p>”; 

eval(’$myvar = ’ . $x . ’;’); 

?>

结果：列出了/ｔｍｐ下面的文件。这个没问题。

来看下一个，翠花上代码：

<?php  

##这段代码来自：[url]https://www.owasp.org/index.php/Direct_Dynamic_Code_Evaluation_%28%27Eval_Injection%27%29[/url] 

$myvar = “varname”;  

$x = $_GET[’arg’];  

echo $x.”<p>”; 

eval(“$myvar = $x;”);  

?>

这段代码，有问题（小菜感觉），  eval(“$myvar = $x;”);  这里应该执行不了吧。

我的回复：

我再改：

<?php

$myvar = “$varname”;  

$x = $_GET[’arg’];  

echo $x.”<p>”; 

eval(“$myvar= $x;”);

?>

回显：成功（就不粘贴dir返回的东西了）

我再改：

<?php

$myvar = “varname”;  

$x = $_GET[’arg’];  

echo $x.”<p>”; 

eval($myvar=$x.’;’);

?>

回显:成功

我再改：

<?php

$myvar = “varname”;  

$x = $_GET[’arg’];  

echo $x.”<p>”; 

eval($x.’;’);

?>

回显：成功（$myvar这个变量都没用了）

下面我就不再改了！你自己弄吧！

你可以在测试的时候输出$x和$myvar这两个变量的值！比较他的变化！还有就是 . “ ’ 这个四个符号在语句中的利用！
